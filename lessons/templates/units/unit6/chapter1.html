We assigned s to a string. In the secound line, we said Python to get the s variable and run a function that it has built-in by using a dot or a period seperator and then a pair of parantheses within which arguments go if required. The following are the functions built-in to a string in Python
  • capitalize(): Capitalize the first letter of the string
  • casefold(): Return a version of the string suitable for caseless comparisons
  • center(): Return a centered string of length width
  • count(): Counts how many times a a character or a string occurs in the called string
  • encode(): Encodes a string to bytes
  • endswith(): Check if the called string ends with the given character
  • expandtabs(): Return a copy where all tab characters are expanded using spaces
  • find(): Finds the given string in the called string
  • format(): Return a formatted version of S, using substitutions from args and kwargs. The substitutions are identified by braces ('{' and '}')
  • format(): Return a formatted version of S, using substitutions from mappings. The substitutions are identified by braces ('{' and '}')
  • index(): Returns the position of a character or group of character in the called string
  • lower(): Converts all letters to lower case
  • replace(): Return a copy with all occurrences of substring old replaced by new
  • startswith(): Check if the called string starts with given chracter
  • split(): Split the called string on the given character and return a list
  • upper(): Converts all letters to upper case
So there are a lot functions that can be called over a string. You can run the dir() function to see all the functions that can be called over a string. One problem that Pythoners face is how to format a string which have numbers in many places. One way is by converting the integer/float to a string using the str() function and then concantating it. But if you have more than a few numbers it gets really tedious. That is why we have string formatting. Let's make for example a discription line for a imaginary person John who is 25 years old, earns $1000 a month, owns 2 cars, has 1 child of 3 years. Now the steps differ from Python 3.6 and above and Python 3.5 and lower. So first let's look for Python 3.5 and lower 
age = 25
salery = 1000
cars = 2
child = 1
child_age = 3
print("John is {} years old and earns ${}. He has {} cars and has {} child who is of {} years".format(age, salery, cars, child, child_age))

# Result
John is 25 years old and earns $1000. He has 2 cars and one has 1 child who is of 3 years
So we used the .format() function over the string and it filled the {} in the order of arguments given to the function. Now let's see the above for Python 3.6 and above, the above is also valid in Python 3.6 and above but it is easier to use something called f-strings which have a f at the starting of the string and instead of curly braces they hold the name of the variable from which the data shoudl be recieved 
age = 25
salery = 1000
cars = 2
child = 1
child_age = 3
print(f"John is {age} years old and earns ${salery}. He has {cars} cars and has {child} child who is of {child_age} years")

# Result
John is 25 years old and earns $1000. He has 2 cars and one has 1 child who is of 3 years
So it did the same thing. Please note that the Python 3.5 syntax is valid in Python 3.6 but it is invalid in vice versa {% endblock %}
