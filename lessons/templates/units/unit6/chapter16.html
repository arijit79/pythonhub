{% extends "base_chapter.html" %} {% block chapter %} Once you have learned to open, read and close files. Now let's modify files. There are two ways one can overwrite a file. First way is to use the write mode and the other is by using append mode. First we will discuss about writing files. In order to write to a file use the w mode instead of r. Then in order to write to a file just use the .write() method over the file. Let's open a file called hello.txt and write Hello From Python Hub to the file 
f = open("hello.txt", "w")
f.write("Hello From Python Hub\n")
f.close()
It is to note that if the file does not exist, the w mode will create the file for you. Also if the file does exist, the w mode will clear out the file for writing. Now the file should be present in the current directory. You can even print its contents by using Python by opening the file in read mode and running the .read() function Note that we have put a \n after the Hub word, this is to make a newline. They are special characters that have certain special meaning. Here are a few more in the table
Special Character Meaning
\n Newline
\t Horizontal Tab
\\ Backslash
\' Single Quote
\" Double Quote
\r Carraige Return
\t Vertical Tab
\f LineFeed
\f FormFeed
Now let's talk about appending files, you can append to a file by using the a mode and running the .write() function over the file. Let's try toopen the same file and write Hello Again to the file 
f = open("hello.txt", "a")
f.write("Hello Again")
f.close()
Now if we try to print the contents of the file we should see both the lines. Also like the w mode if the file does not exist it will create it but will not remove any data if the file already exists. Let's try ro print the contents of the file using the mothod we learned in the previous chapter 
f = open("hello.txt", "r")
print(f.read())
f.close()

# Result
Hello From Python Hub
Hello Again
So as you can see that both of them wrote something to the file. Now that you have learned to write and append, in the next chapter we will look at opening complex file types in Python {% endblock chapter %}
